# 기술 구현 완료 목록 (1/N)

## 파일 개요

이 문서는 현재까지 프로젝트에서 구현이 완료된 주요 기능 및 기술적인 사항들을 상세하게 기록합니다. 프로젝트의 진행 상황을 추적하고 완료된 작업을 명확히 하는 데 목적이 있습니다.

## 완료된 구현 목록

### 1. 프로젝트 초기 설정 및 핵심 구조

*   **기술 스택 기반 구축:**
    *   프레임워크: Next.js 14+ (App Router 기반)
    *   언어: TypeScript
    *   스타일링: TailwindCSS, ShadCN UI
    *   상태 관리: React Hooks (useState, useEffect, useContext 등) 기본 사용 (상태 관리 라이브러리 미도입)
    *   패키지 매니저: `pnpm` 사용
*   **UI 라이브러리 연동:**
    *   ShadCN UI CLI(`npx shadcn@latest add`)를 통해 필요한 컴포넌트 개별 설치 및 프로젝트 내 `components/ui` 폴더에 통합.
    *   현재 연동된 주요 컴포넌트: `Button`, `Card`, `Input`, `Dialog`, `Toast`, `Toaster`, `Badge`, `Label`, `Switch`, `NavigationMenu`, `Sheet`, `Accordion`, `Skeleton`, `RadioGroup`, `DropdownMenu`, `ScrollArea`, `Alert`, `Table`, `Textarea`, `Form`, `Tabs`, `Separator` 등.
*   **기본 Docker 환경 구성:**
    *   `Dockerfile`:
        *   Base 이미지: `node:20-slim` 사용.
        *   의존성 설치: `pnpm install --frozen-lockfile` 사용.
        *   (개선 예정: Multi-stage 빌드를 통한 프로덕션 이미지 최적화 - 현재는 단일 스테이지).
    *   `docker-compose.yml`:
        *   `web` 서비스 (Next.js 앱):
            *   컨테이너 이름: `exam-platform`
            *   환경 변수: `DATABASE_URL`, `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` 등 `.env` 파일 통해 주입.
            *   포트 매핑: 호스트 `3772` -> 컨테이너 `3000` (개발/프로덕션 공통 포트).
            *   볼륨 마운트: `./public/images:/app/public/images` (업로드된 이미지 영구 저장).
            *   네트워크: `ngnet` (외부 생성 네트워크) 사용.
        *   `db` 서비스 (PostgreSQL):
            *   이미지: `postgres:15-alpine`
            *   컨테이너 이름: `exam-db`
            *   환경 변수: `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB` 등 `.env` 파일 통해 주입.
            *   볼륨 마운트: `db_data:/var/lib/postgresql/data` (DB 데이터 영구 저장).
            *   네트워크: `ngnet` 사용.
    *   `.dockerignore`: `node_modules`, `.next`, `dist` 등 빌드/실행에 불필요한 파일 제외.
*   **ORM 및 데이터베이스 설정:**
    *   ORM: Drizzle ORM 사용.
    *   DB 연결: `db/index.ts` 에서 `postgres` 드라이버(`node-postgres`)를 사용하여 PostgreSQL 데이터베이스 연결 설정. `DATABASE_URL` 환경 변수 사용.
    *   **데이터베이스 스키마 정의 (`db/schema/*`):** PostgreSQL 기준.
        *   **`users` 테이블 (`db/schema/index.ts`):** 사용자 정보 관리.
            *   `id` (uuid, PK): 사용자 고유 ID (자동 생성).
            *   `clerkId` (text, notNull, unique): Clerk 인증 시스템의 사용자 ID.
            *   `name` (text, notNull): 사용자 이름.
            *   `email` (text, notNull, unique): 사용자 이메일.
            *   `image` (text, nullable): 사용자 프로필 이미지 URL.
            *   `createdAt` (timestamp, defaultNow, notNull): 생성 시각.
            *   `updatedAt` (timestamp, defaultNow, notNull): 마지막 수정 시각.
        *   **`images` 테이블 (`db/schema/images.ts`):** 업로드된 모든 이미지의 메타데이터 관리.
            *   `id` (serial, PK): 이미지 고유 ID (자동 증가).
            *   `filename` (text, notNull): 서버에 저장된 파일명 (해시값 또는 UUID 기반).
            *   `originalName` (text, notNull): 사용자가 업로드한 원본 파일명.
            *   `path` (text, notNull): 서버 내 저장 경로 (예: `uploaded/images/2024/...`).
            *   `type` (varchar(20), notNull): 이미지 용도 구분 ('question', 'option', 'explanation').
            *   `size` (integer, notNull): 파일 크기 (bytes).
            *   `mimeType` (varchar(50), notNull): 이미지 MIME 타입 (예: 'image/png', 'image/jpeg').
            *   `questionId` (integer, nullable): 이미지가 속한 문제의 ID (외래 키는 설정되지 않음, 추후 필요시 `questions.id` 참조 고려).
            *   `optionId` (integer, nullable): 이미지가 속한 선택지의 인덱스 또는 ID (현재는 인덱스 개념으로 사용될 수 있음).
            *   `createdAt` (timestamp, defaultNow): 생성 시각.
            *   `updatedAt` (timestamp, defaultNow, notNull): 마지막 수정 시각.
            *   `status` (text, default 'active'): 이미지 상태 ('active': 사용 중, 'pending_deletion': 삭제 예정, 'deleted': 삭제됨 - Soft Delete 개념).
            *   `lastCheckedAt` (timestamp, defaultNow): (용도 미정확) 마지막 확인 시각.
            *   `imageStatusEnum` (pgEnum): `status` 필드 값 제한을 위한 Enum 정의.
        *   **`questions` 테이블 (`db/schema/questions.ts`):** 핵심 문제 데이터 저장.
            *   `id` (uuid, PK): 문제 고유 ID (자동 생성).
            *   `content` (text, notNull): 문제 본문 텍스트 (HTML 포함 가능).
            *   `options` (jsonb, notNull): 객관식 선택지 배열.
                *   배열 요소 타입: `{ number: number; text: string; images: { url: string; hash: string }[] }`
                *   `number`: 선택지 번호 (사용자 표시용).
                *   `text`: 선택지 텍스트.
                *   `images`: 해당 선택지에 첨부된 이미지 정보 배열 (`url`: 저장 경로, `hash`: 파일 해시).
            *   `answer` (integer, notNull): 정답 선택지의 **0-based 인덱스**.
            *   `explanation` (text, nullable): 문제 해설 텍스트 (HTML 포함 가능).
            *   `tags` (jsonb, default `[]`): 문제 분류를 위한 태그 문자열 배열 (예: ["시험명:정보처리기사", "년도:2023", "회차:1", "과목:데이터베이스"]).
            *   `images` (jsonb, default `[]`): 문제 본문에 첨부된 이미지 정보 배열.
                *   배열 요소 타입: `{ url: string; hash: string }` (`url`: 저장 경로, `hash`: 파일 해시).
            *   `explanationImages` (jsonb, default `[]`): 해설에 첨부된 이미지 정보 배열.
                *   배열 요소 타입: `{ url: string; hash: string }` (`url`: 저장 경로, `hash`: 파일 해시).
            *   `userId` (text, nullable): 문제 생성자의 ID (현재 Clerk ID 사용 추정, `users` 테이블 참조 미설정).
            *   `createdAt` (timestamp, defaultNow, notNull): 생성 시각.
            *   `updatedAt` (timestamp, defaultNow, notNull): 마지막 수정 시각.
        *   **`examResults` 테이블 (`db/schema/index.ts`):** 시험 응시 결과 요약 정보 (기본 구조만 구현, 상세화 예정).
            *   `id` (uuid, PK): 결과 고유 ID (자동 생성).
            *   `userId` (uuid, FK -> users.id): 시험 응시자의 ID (CASCADE 삭제 옵션).
            *   `examId` (uuid, nullable): 응시한 시험의 ID (현재 `exams` 테이블 미사용으로 FK 설정 없음, 추후 연동 필요).
            *   `score` (integer, notNull): 최종 점수.
            *   `totalQuestions` (integer, notNull): 전체 문항 수.
            *   `correctAnswers` (integer, notNull): 맞힌 문항 수.
            *   `wrongAnswers` (jsonb, notNull): 틀린 문제의 **0-based 인덱스** 배열 (응시 문제 목록 기준).
            *   `completedAt` (timestamp, notNull): 시험 완료 시각.
            *   `createdAt` (timestamp, defaultNow, notNull): 결과 레코드 생성 시각.
        *   **(미사용 또는 레거시)** `imageHistory`, `exams`, `questionsLegacy`.
    *   **마이그레이션:** Drizzle Kit (`drizzle-kit`) CLI 도구 사용.
        *   스키마 변경 감지 및 마이그레이션 SQL 파일 생성: `pnpm db:generate` (내부적으로 `drizzle-kit generate:pg` 실행).
        *   생성된 SQL 파일을 데이터베이스에 적용: `pnpm db:migrate` (내부적으로 `drizzle-kit migrate` 실행).
*   **기본 폴더 구조 및 라우팅:**
    *   **App Router 기반 폴더 구조:** 기능별 최상위 폴더(`learn`, `manage`, `test`, `community`, `profile`, `guide`) 사용.
        *   학습: `/learn/exams`, `/learn/exams/[examName]`, `/learn/exams/[examName]/[year]/[session]`
        *   문제 관리: `/manage/questions`, `/manage/questions/new`, `/manage/questions/edit/[questionId]`
        *   모의고사 선택: `/test/select/exam`, `/test/select/exam/[examName]` (페이지 틀만 생성)
    *   **컴포넌트 구조:** `components/ui` (ShadCN), `components/common` (공통 UI - `Breadcrumb`, `CommonImage`, `ImageZoomModal`, `LoadingSpinner`), `components/layout` (레이아웃 - `Navbar`, `Footer`), `components/manage` (문제 관리 관련 - `ManualForm`, `PasteForm`, `ImageUploadArea`), `components/study` (학습 페이지 관련 - `StudyPageHeader`), `components/exam-selection` (시험/회차 선택 모듈 - `ExamListDisplay`, `ExamSessionListDisplay`).
    *   기타 폴더: `db` (DB 연결 및 스키마), `types` (전역 타입 정의 - `index.ts`, `IQuestion` 등), `utils` (유틸리티 함수 - `image.ts`, `questionParser.ts`), `public` (정적 파일 - 이미지 등), `styles` (전역 CSS).
*   **기본 미들웨어 설정 (`middleware.ts`):**
    *   현재는 모든 경로 접근을 허용하는 임시 설정 (`return NextResponse.next()`). Clerk 인증 연동은 예정 상태.
    *   `matcher` 설정: 정적 파일(`.*\..*`), `_next` 폴더 제외한 대부분의 경로 및 API 경로(`/(api|trpc)(.*)`)를 포함하여 미들웨어가 실행될 경로 지정.
