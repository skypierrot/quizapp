# 기술 구현 완료 목록 (2/N)

## 완료된 구현 목록 (계속)

### 2. 인증 시스템 (Clerk)

*   **기본 연동:**
    *   Clerk Provider (`<ClerkProvider>`)를 `app/layout.tsx`에 적용하여 전역적으로 Clerk 컨텍스트 제공.
    *   Clerk 관련 환경 변수 (`NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY`)를 `.env` 파일에서 관리하고 `docker-compose.yml`을 통해 주입.
*   **미들웨어 설정 (`middleware.ts`):**
    *   현재는 모든 경로 접근을 허용하는 임시 설정 상태입니다 (`NextResponse.next()`).
    *   `clerkMiddleware()`를 사용하여 인증 로직을 구현할 준비가 되어 있습니다. `createRouteMatcher`를 이용해 공개 경로(로그인, 회원가입)를 정의하고, 나머지 경로는 `auth().protect()`로 보호하는 표준 패턴 적용 예정입니다.
    *   `config.matcher`를 통해 미들웨어가 적용될 경로를 지정했습니다 (정적 파일 및 `_next` 폴더 제외).
*   **UI 컴포넌트:**
    *   로그인/로그아웃 상태에 따라 UI를 다르게 표시하기 위해 `<SignedIn>`, `<SignedOut>`, `<UserButton>` 컴포넌트를 `Navbar.tsx`에 적용했습니다.
*   **사용자 정보 동기화 (부분 구현):**
    *   로그인 시 Clerk 사용자 정보를 기반으로 `users` 테이블에 사용자 데이터를 생성하거나 업데이트하는 로직은 아직 명시적으로 구현되지 않았습니다. (API 레벨에서 필요시 `clerkClient` 사용 준비).
*   **제약사항:** 현재 미들웨어에서 인증 로직이 활성화되지 않아 모든 페이지가 비로그인 상태에서도 접근 가능합니다.

### 3. 문제 관리 기능 (`/manage/questions`)

*   **문제 생성:**
    *   **수동 입력 (`/manage/questions/new`, `components/manage/ManualForm.tsx`):**
        *   ShadCN `Form` 컴포넌트와 `react-hook-form`, `zod`를 사용하여 폼 유효성 검사 및 상태 관리.
        *   문제 내용, 선택지(동적 추가/삭제 가능), 정답(RadioGroup), 해설을 입력받습니다.
        *   `ImageUploadArea` 컴포넌트를 통해 문제, 선택지, 해설에 이미지 첨부 가능 (클라이언트 해시 계산, 임시 업로드, 최종 저장 API 연동).
        *   태그 입력: 시험명, 년도, 회차, 과목 등을 직접 입력하여 `tags` 배열로 저장 (현재는 단순 문자열 입력, 향후 드롭다운 또는 자동완성 개선 가능).
        *   API 호출: `/api/questions` (POST)를 호출하여 단일 문제 저장.
    *   **복사/붙여넣기 입력 (`/manage/questions/new`, `components/manage/PasteForm.tsx`):**
        *   대량의 텍스트를 `Textarea`에 붙여넣으면, 정의된 규칙(`utils/questionParser.ts`의 `parseQuestionsFromText`)에 따라 문제, 선택지, 정답, 해설을 자동으로 파싱하여 여러 문제를 동시에 생성합니다.
        *   파싱 로직:
            *   문제 구분자 (예: "문제", "문."), 선택지 구분자 (예: ①, ②, ③, ④ 또는 1., 2., 3., 4.), 정답 구분자 (예: "정답:", "답:"), 해설 구분자 (예: "해설:")를 기준으로 텍스트를 분리합니다.
            *   파싱된 데이터는 내부적으로 `ManualForm`과 유사한 구조로 변환되어 처리됩니다.
        *   이미지 처리: 텍스트와 함께 붙여넣기된 이미지(클립보드 이미지) 또는 Drag & Drop된 이미지를 처리합니다.
            *   클립보드 이미지: `handlePaste` 이벤트에서 `event.clipboardData.files`를 확인하여 이미지 파일 처리.
            *   Drag & Drop: `ImageUploadArea`를 통해 처리.
            *   클라이언트 해시 계산 및 임시 업로드 API (`/api/images/upload`) 호출.
        *   API 호출: `/api/questions/batch` (POST)를 호출하여 여러 문제를 한 번에 저장 (`FormData` 사용).
*   **문제 목록 조회 (`/manage/questions/list`, 이전 `/questions/list`):**
    *   API 호출: `/api/questions` (GET)를 호출하여 문제 목록을 가져옵니다. 페이지네이션 파라미터(`page`, `limit`) 사용.
    *   **표시 내용:** 문제 ID, 문제 내용 요약, 태그, 생성/수정 시각.
    *   **페이지네이션:** 하단에 페이지 번호 및 이전/다음 버튼 제공 (`usePagination` 훅 또는 유사 로직 사용 예상).
    *   **요약/상세 보기 토글:** 각 문제 행에 토글 버튼을 두어 문제 내용 전체, 선택지, 정답, 해설을 펼쳐볼 수 있는 기능 (구현됨).
    *   **이미지 표시:** 문제, 선택지, 해설에 포함된 이미지를 `CommonImage` 컴포넌트를 사용하여 표시 (로딩 상태, 에러 처리, 비율 유지 기능 포함).
    *   **해설 보기:** 별도의 버튼 또는 토글 영역을 통해 해설 표시. 필요시 `Dialog` 컴포넌트 사용.
    *   **이미지 확대:** 표시된 이미지를 클릭하면 `ImageZoomModal`을 통해 원본 크기 또는 확대된 이미지 보기 기능 (`useImageZoom` 훅 사용).
    *   **문제 삭제:** 각 문제 행에 삭제 버튼 제공. 클릭 시 확인 절차 후 `/api/questions/[questionId]` (DELETE) API 호출 (Soft Delete 또는 Hard Delete 방식은 API 구현에 따름).
    *   **(미구현)** 검색 및 필터링: 태그, 내용 검색 등 상세 필터 기능은 아직 구현되지 않음.
*   **문제 수정 (`/manage/questions/edit/[questionId]`):**
    *   API 호출: `/api/questions/[questionId]` (GET)를 호출하여 수정할 문제의 데이터를 가져옵니다.
    *   데이터 로딩: 가져온 데이터를 `ManualForm`의 초기값으로 설정하여 사용자에게 기존 내용을 보여줍니다.
    *   수정 로직: `ManualForm`을 사용하여 내용 수정 (이미지 추가/삭제 포함).
    *   API 호출: 수정 완료 시 `/api/questions/[questionId]` (PUT 또는 PATCH) API를 호출하여 변경 사항 저장.

### 4. 이미지 처리 시스템

*   **클라이언트 측 처리 (`utils/image.ts`):**
    *   `getFileHash`: 이미지 파일(`File` 객체)을 입력받아 `crypto.subtle.digest('SHA-256')`를 사용하여 클라이언트 사이드에서 파일의 해시값(SHA-256)을 계산합니다. 이는 서버 업로드 전 중복 검사 등에 활용될 수 있습니다.
    *   `isValidImage`: 파일의 MIME 타입과 크기를 기준으로 업로드 가능한 이미지인지 검사하는 유틸리티 함수.
*   **업로드 컴포넌트 (`components/manage/ImageUploadArea.tsx`):**
    *   Drag & Drop 및 파일 선택 버튼을 통한 이미지 입력 인터페이스 제공.
    *   업로드된 이미지 미리보기 표시 (삭제 버튼 포함).
    *   이미지 파일 유효성 검사 (`isValidImage` 활용).
    *   파일 해시 계산 (`getFileHash` 활용).
    *   임시 업로드 API (`/api/images/upload`) 호출: 계산된 해시값과 함께 이미지 파일을 서버로 전송. 서버는 이미지를 임시 폴더(예: `public/tmp/images`)에 저장하고 임시 경로와 해시 반환.
*   **서버 측 처리 (API):**
    *   **임시 업로드 (`/api/images/upload` - POST):**
        *   `multipart/form-data` 형식으로 이미지 파일과 해시값 수신.
        *   수신된 파일을 서버의 임시 저장 경로 (`public/tmp/images`)에 저장.
        *   성공 시 저장된 임시 파일 경로 및 해시 반환.
        *   (개선점: 동일 해시 파일 중복 저장 방지 로직 추가 가능).
    *   **최종 저장 (`/api/images/finalize` - POST):**
        *   문제/선택지/해설 등과 함께 최종 저장될 이미지들의 임시 경로 또는 해시 리스트를 수신.
        *   임시 폴더의 해당 이미지 파일들을 최종 저장 경로(예: `public/uploaded/images/[년도]/[월]/[파일명]`)로 이동 또는 복사.
        *   `images` 테이블에 이미지 메타데이터(파일명, 원본명, 경로, 타입, 크기, MIME 타입, 관련 문제/선택지 ID 등) 저장.
        *   `questions` 테이블의 해당 문제 레코드에 이미지 정보(최종 URL, 해시) 업데이트.
        *   성공 시 최종 이미지 URL 목록 반환.
        *   (구현 상세: 트랜잭션 처리, 임시 파일 자동 정리 로직 필요).
    *   **이미지 제공:** Next.js의 `public` 폴더 기능을 통해 `/uploaded/images/...` 경로로 이미지 파일 직접 서빙.
*   **공통 이미지 컴포넌트 (`components/common/CommonImage.tsx`):**
    *   Next.js `Image` 컴포넌트를 래핑하여 사용.
    *   `src` prop으로 이미지 URL을 받습니다.
    *   로딩 상태 표시 (예: 스켈레톤 UI).
    *   이미지 로드 실패 시 대체 이미지 또는 에러 메시지 표시.
    *   `maintainAspectRatio` prop을 통해 이미지 비율 유지 여부 제어.
    *   `alt` 텍스트 자동 생성 또는 전달받아 설정 (접근성).
    *   ShadCN 스타일 또는 커스텀 스타일 적용.

### 5. 학습 기능 (`/learn/exams`)

*   **시험 과목 목록 (`/learn/exams`):**
    *   API 호출: `/api/exam-instances?distinct=examName` (또는 유사 API)를 호출하여 DB에 저장된 고유한 시험 과목명 목록을 가져옵니다.
    *   표시: 가져온 시험 과목명을 리스트 또는 카드 형태로 표시합니다 (`ExamListDisplay` 컴포넌트 사용).
    *   링크: 각 과목명 클릭 시 해당 과목의 회차 목록 페이지(`/learn/exams/[examName]`)로 이동합니다.
    *   검색: 상단 검색창을 통해 과목명 필터링 기능 제공.
*   **시험 회차 목록 (`/learn/exams/[examName]`):**
    *   API 호출: `/api/exam-instances?examName=[examName]` (또는 유사 API)를 호출하여 특정 시험 과목에 해당하는 년도 및 회차 목록을 가져옵니다.
    *   표시: 년도별로 그룹화하여 회차 정보를 리스트 또는 아코디언 형태로 표시합니다 (`ExamSessionListDisplay` 컴포넌트 사용).
    *   링크: 각 회차 클릭 시 해당 회차의 문제 학습 페이지(`/learn/exams/[examName]/[year]/[session]`)로 이동합니다.
    *   검색: 상단 검색창을 통해 년도 또는 회차 정보 필터링 기능 제공.
    *   Breadcrumb: 상단에 현재 위치(예: 홈 > 학습 > 정보처리기사)를 나타내는 Breadcrumb 표시 (`components/common/Breadcrumb.tsx`).
*   **문제 학습 페이지 (`/learn/exams/[examName]/[year]/[session]`):**
    *   API 호출: `/api/questions?examName=[examName]&year=[year]&session=[session]` (또는 태그 기반 필터링) API를 호출하여 해당 회차의 문제 목록 전체를 가져옵니다.
    *   **헤더 영역 (`components/study/StudyPageHeader.tsx`):**
        *   현재 시험 정보(과목, 년도, 회차), 문제 수 표시.
        *   **한 문제씩 보기 토글:** 전체 문제 스크롤 방식과 한 문제씩 넘겨보는 방식 간 전환 (`Button` 또는 `Switch` 사용).
        *   **선택지 섞기 토글:** 문제의 선택지 순서를 무작위로 섞는 기능 (`shuffleArray` 유틸리티 함수 및 `useMemo` 활용).
        *   **전체 정답/해설 보기 토글:** 모든 문제의 정답과 해설을 한 번에 보여주거나 숨기는 기능.
        *   Breadcrumb 표시.
    *   **문제 표시 영역:**
        *   가져온 문제 데이터를 순회하며 각 문제 표시.
        *   **문제 본문:** `content` 필드 표시 (HTML 렌더링 지원 필요 시 `dangerouslySetInnerHTML` 또는 sanitizer 라이브러리 사용 고려). 본문에 포함된 이미지는 `CommonImage`로 표시.
        *   **선택지:** `options` 배열을 순회하며 각 선택지 번호와 텍스트 표시. 선택지에 포함된 이미지는 `CommonImage`로 표시. 사용자가 선택지를 클릭하면 선택한 상태를 시각적으로 표시 (예: 배경색 변경).
        *   **정답 확인:** 사용자가 선택지를 고른 후 또는 '정답 보기' 버튼 클릭 시, 정답 선택지에 O 또는 X 표시, 또는 정답 강조 표시.
        *   **해설:** '해설 보기' 버튼 클릭 또는 토글 시 `explanation` 필드 내용 표시. 해설에 포함된 이미지는 `CommonImage`로 표시.
        *   **개별 정답/해설 토글:** 각 문제별로 정답 및 해설을 보거나 숨길 수 있는 버튼 제공.
    *   **한 문제씩 보기 모드:**
        *   활성화 시 한 번에 한 문제만 화면에 표시.
        *   하단 또는 양옆에 이전/다음 문제 이동 버튼 제공.
        *   키보드 네비게이션 지원 (예: 좌우 화살표 키로 문제 이동).
        *   현재 문제 번호 및 전체 문제 수 표시 (예: 3 / 20).
    *   **상태 관리:** 현재 문제 인덱스, 사용자 선택 답안, 정답/해설 표시 상태 등을 `useState` 또는 `useReducer`를 사용하여 관리.
    *   **이미지 확대:** 문제, 선택지, 해설에 포함된 이미지 클릭 시 `ImageZoomModal` 표시.
