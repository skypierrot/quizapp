## 완료된 구현 목록 (계속)

### 6. 모의고사 기능 (기본 구조)

*   **시험 선택 (`/test/select/exam`, `/test/select/exam/[examName]`):**
    *   UI/UX: `/learn/exams` 경로와 동일한 구조 및 컴포넌트 (`ExamListDisplay`, `ExamSessionListDisplay`)를 재사용하여 구현.
    *   목표: 사용자가 특정 시험 과목 및 회차를 선택하여 모의고사를 시작할 수 있도록 하는 진입점.
    *   상태: 현재 페이지는 시험/회차 선택 UI만 구현되어 있으며, 실제 모의고사 시작 로직 및 시험 응시 화면(`TestPage`) 연동은 미구현 상태.
    *   링크: 각 회차 선택 시 아직 실제 시험 페이지(`/test/start/[...params]`)로 연결되지 않음 (추후 구현 필요).

### 7. UI/UX 개선 및 리팩토링

*   **URL 구조 재구성:**
    *   기존의 `/bank`(문제 관리), `/study`(학습), `/questions`(문제 목록 - 초기) 등의 경로를 사용자 기능 중심으로 `/manage`(관리), `/learn`(학습), `/test`(모의고사)로 명확하게 분리하고 재구성했습니다.
    *   관련 폴더 구조(`app` 디렉토리 하위) 변경 및 코드 내 모든 `Link` 컴포넌트와 리다이렉션 경로를 수정했습니다.
    *   이 과정에서 발생할 수 있는 404 오류 방지를 위해 경로 일관성을 확인했습니다.
*   **컴포넌트 모듈화 및 재사용:**
    *   **시험/회차 선택 UI 분리:** `/learn/exams/...` 와 `/test/select/exam/...` 경로에서 중복 사용되던 시험 과목 목록 및 회차 목록 표시 로직을 별도의 공통 컴포넌트로 추출했습니다.
        *   `components/exam-selection/ExamListDisplay.tsx`: 시험 과목 목록 표시 (검색 기능 포함).
        *   `components/exam-selection/ExamSessionListDisplay.tsx`: 특정 과목의 년도/회차 목록 표시 (검색 기능, 아코디언 UI 포함).
    *   각 페이지 컴포넌트(`app/learn/exams/page.tsx`, `app/learn/exams/[examName]/page.tsx`, `app/test/select/exam/page.tsx`, `app/test/select/exam/[examName]/page.tsx`)에서는 이 공통 컴포넌트들을 호출하고, `basePath` 등의 props를 전달하여 각 경로에 맞게 링크 URL을 동적으로 생성하도록 구현했습니다.
*   **공통 UI 컴포넌트 활용:**
    *   `components/common/CommonImage.tsx`: 이미지 로딩, 에러 처리, 비율 유지 등 이미지 관련 공통 로직 처리.
    *   `components/common/ImageZoomModal.tsx` & `hooks/useImageZoom.ts`: 이미지 클릭 시 확대 보기 기능 제공.
    *   `components/common/Breadcrumb.tsx`: 페이지 상단에 현재 경로 표시.
    *   `components/common/LoadingSpinner.tsx`: 데이터 로딩 중 상태 표시.
    *   ShadCN UI 컴포넌트(`Button`, `Card`, `Dialog`, `Toast`, `Input` 등)를 일관되게 사용하여 전체적인 UI 통일성 확보.
*   **이미지 표시 일관성:** `CommonImage`의 `maintainAspectRatio` prop을 활용하여 문제 목록, 학습 페이지 등 다양한 위치에서 이미지가 의도치 않게 늘어나거나 줄어들지 않고 원본 비율을 유지하도록 개선했습니다.
*   **모바일 메뉴 개선 (`components/layout/Navbar.tsx`):**
    *   모바일 화면에서 `Sheet` 컴포넌트로 표시되는 메뉴 항목 클릭 시, 메뉴가 자동으로 닫히도록 각 메뉴 `Link`에 `onClick` 핸들러를 추가하여 `setOpen(false)`를 호출하도록 수정했습니다.
*   **기타 개선:**
    *   리스트 렌더링 시 고유 `key` prop 누락으로 인한 React 경고 수정 (예: 선택지 렌더링).
    *   `StudyPageHeader`의 '선택지 섞기' 토글 UI를 `Switch`에서 `Button`으로 변경하여 다른 토글 버튼들과의 시각적 일관성 확보.

### 8. 서버 API 엔드포인트 (구현된 항목)

*   **문제 관리 (`app/api/questions`):**
    *   `GET /api/questions`: 문제 목록 조회 (페이지네이션 `page`, `limit` 지원, 태그 기반 필터링 `examName`, `year`, `session` 지원).
    *   `POST /api/questions`: 단일 문제 생성 (`ManualForm`에서 사용).
    *   `GET /api/questions/[questionId]`: 특정 문제 상세 정보 조회.
    *   `PUT /api/questions/[questionId]`: 특정 문제 정보 수정 (`ManualForm`에서 사용).
    *   `DELETE /api/questions/[questionId]`: 특정 문제 삭제 (Soft Delete 또는 Hard Delete).
    *   `POST /api/questions/batch`: 여러 문제 동시 생성 (`PasteForm`에서 사용, 현재 `FormData` 처리 로직 필요).
*   **이미지 처리 (`app/api/images`):**
    *   `POST /api/images/upload`: 이미지 임시 업로드 (파일과 해시 수신, 임시 경로 반환).
    *   `POST /api/images/finalize`: 임시 이미지 최종 저장 (임시 경로/해시 리스트 수신, 최종 경로 이동 및 DB 메타데이터 저장).
*   **시험/회차 정보 (`app/api/exam-instances`):**
    *   `GET /api/exam-instances`: 시험 인스턴스(과목, 년도, 회차) 목록 조회. `distinct=examName` 쿼리 파라미터를 통해 고유 시험 과목명 목록 조회 지원. `examName=[name]` 파라미터를 통해 특정 과목의 회차 목록 조회 지원.
    *   (참고: `exam-instances`는 실제 DB 테이블이 아니라 `questions` 테이블의 `tags` 정보를 기반으로 동적으로 생성되는 데이터일 수 있음).
*   **API 구현 특징:**
    *   Next.js Route Handler 사용.
    *   Drizzle ORM을 사용하여 데이터베이스 작업 수행.
    *   기본적인 에러 처리 (`try...catch`) 및 `NextResponse.json`을 이용한 응답 반환.
    *   인증/인가 로직은 아직 미적용 상태 (Clerk 연동 필요).
