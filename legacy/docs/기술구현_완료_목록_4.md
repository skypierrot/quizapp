## 완료된 구현 목록 (계속)

### 9. 유틸리티 함수 (`utils/`)

*   **`utils/image.ts`:**
    *   `getFileHash(file: File): Promise<string>`: 클라이언트 사이드에서 이미지 파일의 SHA-256 해시를 비동기적으로 계산합니다. Web Crypto API (`crypto.subtle.digest`) 사용.
    *   `isValidImage(file: File): boolean`: 파일의 MIME 타입 (`image/*`)과 설정된 최대 크기(예: 5MB)를 기준으로 이미지 파일의 유효성을 검사합니다.
    *   `normalizeImages(images: any): { url: string; hash: string }[]`: API 응답 등에서 다양한 형태(문자열 URL, 객체 배열 등)로 올 수 있는 이미지 정보를 `{ url: string; hash: string }` 형태의 배열로 정규화합니다. (문제 목록, 수정 등 데이터 로딩 시 사용됨)
*   **`utils/questionParser.ts`:**
    *   `parseQuestionsFromText(text: string): Partial<IQuestion>[]`: 텍스트를 입력받아 정규식을 사용하여 문제, 선택지(번호 또는 기호), 정답, 해설을 파싱하고, `IQuestion` 타입의 부분 객체 배열로 반환합니다. 이미지 파싱 로직은 포함되어 있지 않습니다. (`PasteForm`에서 사용)
*   **`utils/cn.ts`:**
    *   `cn(...inputs: ClassValue[]): string`: `clsx`와 `tailwind-merge` 라이브러리를 결합하여 TailwindCSS 클래스 문자열을 조건부로 병합하고 중복/충돌되는 클래스를 정리해주는 유틸리티 함수입니다. (ShadCN UI 기본 제공)
*   **`utils/helpers.ts` (또는 유사 파일):**
    *   `shuffleArray<T>(array: T[]): T[]`: 제네릭 타입을 사용하여 입력된 배열의 요소 순서를 무작위로 섞어 새로운 배열을 반환합니다. (문제 학습 페이지의 선택지 섞기 기능에 사용)
    *   `decodeAndSanitize(text: string | null | undefined): string`: URL 인코딩된 문자열을 디코드하고, 기본적인 XSS 방지를 위한 간단한 정제 로직(옵션)을 포함할 수 있는 함수입니다. (경로 파라미터 처리 등에 사용)

### 10. 커스텀 훅 (`hooks/`)

*   **`hooks/useImageZoom.ts`:**
    *   이미지 확대 모달의 상태(`isOpen`, `imageUrl`)와 모달을 열고 닫는 함수(`openZoom`, `closeZoom`)를 관리하는 커스텀 훅입니다.
    *   `ImageZoomModal` 컴포넌트와 함께 사용됩니다.
*   **`hooks/useToast.ts`:**
    *   ShadCN `Toast` 컴포넌트를 보다 쉽게 사용하기 위한 커스텀 훅입니다. `toast()` 함수를 반환하며, 이 함수를 호출하여 다양한 유형(default, destructive 등)의 토스트 메시지를 표시할 수 있습니다. (`Toaster` 컴포넌트가 레이아웃에 필요)
*   **(사용 가능성)** `usePagination`: 페이지네이션 로직(현재 페이지, 총 페이지 수, 페이지 변경 함수 등)을 관리하는 커스텀 훅 (현재는 각 페이지 컴포넌트 내에서 직접 구현된 경우가 많음).

### 11. 상태 관리

*   **기본 전략:** 전역 상태 관리 라이브러리(Redux, Zustand 등)는 아직 도입하지 않았습니다.
*   **로컬 상태:** 각 컴포넌트의 UI 상태, 폼 입력 값 등은 `useState`를 주로 사용하여 관리합니다.
*   **서버 상태 캐싱/관리:** React Query, SWR 등의 라이브러리는 사용하지 않고, `useEffect` 내에서 `fetch` API를 직접 호출하여 데이터를 가져오고 `useState`로 관리하는 방식을 사용합니다. 이는 데이터 재검증(revalidation), 캐싱, 로딩/에러 상태 관리 등에서 수동적인 구현이 필요함을 의미합니다.
*   **컴포넌트 간 상태 공유:** 주로 Props Drilling 방식을 사용합니다. 상태 공유가 복잡해지는 경우 React Context API 사용을 고려할 수 있으나, 현재 명시적으로 사용된 예는 제한적입니다 (예: `useImageZoom`의 내부 구현 가능성).

### 12. 에러 처리 및 알림

*   **클라이언트 측:**
    *   API 호출 실패, 폼 유효성 검사 실패 등의 경우 `hooks/useToast`를 사용하여 사용자에게 피드백 메시지를 표시합니다.
    *   `try...catch` 구문을 사용하여 비동기 작업(API 호출 등) 중 발생할 수 있는 예외를 처리합니다.
    *   컴포넌트 렌더링 오류는 Next.js의 기본 에러 바운더리 또는 커스텀 에러 페이지(`error.tsx`)를 통해 처리될 수 있습니다 (현재 커스텀 에러 페이지는 기본 설정).
*   **서버 측 (API):**
    *   Route Handler 내에서 `try...catch`를 사용하여 데이터베이스 오류, 요청 처리 중 예외 등을 처리합니다.
    *   오류 발생 시 `NextResponse.json`을 사용하여 적절한 상태 코드(예: 400, 404, 500)와 오류 메시지를 포함한 JSON 응답을 반환합니다.
    *   상세한 서버 로깅은 아직 구현되지 않았습니다.

    *   `any` 타입 사용을 최소화하고 구체적인 타입을 정의하려 노력합니다.
*   **코딩 스타일:** ESLint와 Prettier를 사용하여 코드 포맷팅 및 잠재적 오류 검사를 수행합니다. `.eslintrc.json`, `.prettierrc.js` 설정 파일을 통해 규칙을 관리합니다.
    *   VS Code 설정 (`.vscode/settings.json`)을 통해 저장 시 자동 포맷팅(`editor.formatOnSave`) 및 ESLint 자동 수정(`source.fixAll.eslint`)을 활성화하여 코드 스타일 일관성을 유지합니다.

*   **Git 관리:**
    *   커밋 메시지는 지정된 형식 (`<type>: <subject>`)을 따릅니다 (feat, fix, docs, style, refactor, test, chore 타입 사용).

    *   기능 개발 시 브랜치를 사용하고 Pull Request를 통한 코드 리뷰 프로세스를 권장합니다 (현재는 단독 개발 환경).

### 14. 알려진 제약 사항 및 개선 영역

*   **인증:** Clerk 미들웨어가 활성화되지 않아 현재 모든 페이지 접근이 가능합니다. 사용자 역할(관리자, 일반 사용자) 기반 접근 제어 로직도 미구현 상태입니다.
*   **API 보안:** API 엔드포인트에 대한 인증/인가 처리가 미흡합니다.
*   **오류 처리:** 전반적인 오류 처리 로직이 기본적인 수준이며, 사용자 경험 및 디버깅을 위한 상세한 처리가 부족합니다.
*   **상태 관리:** 복잡한 페이지(문제 학습 페이지 등)에서 `useState` 기반 상태 관리가 비대해질 수 있으며, 서버 상태 관리 라이브러리 도입 시 성능 및 개발 편의성 개선 여지가 있습니다.
*   **테스트:** 단위 테스트, 통합 테스트, E2E 테스트 코드가 부재합니다.
*   **`PasteForm` API 연동:** `/api/questions/batch` API가 현재 JSON 페이로드만 처리 가능하여 `FormData`로 이미지와 함께 전송 시 오류가 발생합니다. API 수정이 필요합니다.
*   **DB 스키마:** `images` 테이블과 `questions` 테이블 간의 관계가 명시적인 외래 키 제약 조건으로 설정되어 있지 않아 데이터 무결성 보장이 약합니다. `examResults` 테이블도 `exams` 테이블과의 관계 설정이 필요합니다.
*   **성능 최적화:** 대량 데이터 처리(문제 목록 등) 시 성능 저하 가능성이 있으며, 코드 스플리팅, 이미지 최적화, DB 쿼리 최적화 등이 필요할 수 있습니다.
*   **모의고사 기능:** 핵심 로직(시험 진행, 시간 측정, 채점, 결과 저장)이 미구현 상태입니다.
